# COMP 5700 – AI Pull Request Security Analysis Project

This repository contains the code and results for Tasks 1–5 of the COMP 5700 course project.  
The objective of this project is to analyze a dataset of pull requests generated by agentic AI tools and determine which tasks are security-related.

## Repository Structure

├── src/ # Source code for all tasks
│ ├── task1.py
│ ├── task2.py
│ ├── task3.py
│ ├── task4.py
│ └── task5.py
│
├── output/ # Generated CSV output files
│ ├── task1_output.csv (Git LFS pointer - too large for GitHub)
│ ├── task2_output.csv
│ ├── task3_output.csv
│ ├── task4_output.csv (Git LFS pointer - too large for GitHub)
│ └── task5_output.csv
│
├── data/ # Parquet input files (not tracked)
├── README.md
└── requirements.txt # Python dependencies

## Tasks Included

### Task 1
Extract pull request metadata from `all_pull_request.csv`.

### Task 2
Extract repository metadata from `all_repository.csv`.

### Task 3
Extract task-type metadata from `pr_task_type.csv`.

### Task 4
Extract commit details from `pr_commit_details.csv`.

### Task 5
Combine processed data from Tasks 1–4 into a unified CSV with:
- ID  
- AGENT  
- TYPE  
- CONFIDENCE  
- SECURITY (boolean flag)

## How to Run

python3 src/task1.py
python3 src/task2.py
python3 src/task3.py
python3 src/task4.py
python3 src/task5.py

Outputs will appear in the `output/` folder.
## Large Output Files (Task 1 & Task 4)

The CSV outputs for Task 1 and Task 4 exceed GitHub’s 100MB per-file limit:

- task1_output.csv ≈ 594 MB
- task4_output.csv ≈ 1.6 GB

Because GitHub cannot store files of this size, these files are tracked using 
Git Large File Storage (Git LFS). GitHub displays LFS pointers instead of the 
actual CSV content.

The full CSVs can be regenerated locally by placing the parquet dataset files 
into the `data/` directory and running:

    python src/task1.py
    python src/task4.py

All other task outputs (Task 2, 3, 5) are included normally in the repository.
